# BMS 故障模拟装置 —— 软件架构说明

基于 **Zephyr RTOS** 的 T2 workspace 应用,由原 STM32 CubeIDE 工程(`D:\DESKTOP\Zhiyun\ZhiYun`)移植而来。
设计目标:**换 MCU、换 DAC 时,应用业务层代码零改动**。

## 1. 移植策略

原工程为 STM32H743 + FreeRTOS + HAL。移植到 Zephyr 时:

| 原工程 | Zephyr 中的落地 |
|---|---|
| `App/`(业务逻辑) | `app/` —— 接口去 HAL 化(`HAL_StatusTypeDef`→`int`,移除 `main.h`),`.c` 由原工程搬运填充 |
| `BSP/`(bsp_dac81416 / bsp_can / bsp_eth / bsp_sram) | `drivers/` —— 重写为 Zephyr out-of-tree 驱动,SPI/GPIO/CAN 经 devicetree |
| `Core/`(CubeMX 生成的 HAL/MSP/时钟) | 由 Zephyr devicetree + SoC HAL(west 拉取 hal_stm32)取代,不再手工维护 |
| FreeRTOS | Zephyr 内核 |

## 2. 目录结构

```

├── west.yml              # 依赖清单(锁 zephyr + hal_stm32);换 MCU 改此处 HAL
├── CMakeLists.txt        # 顶层:注册 boards/drivers/dts 为 out-of-tree 资源
├── Kconfig / prj.conf    # 项目级配置项与默认开关
│
├── app/                  ★应用业务层(接口已去 HAL 化)
│   ├── include/app/      各模块对外头文件
│   │   ├── voltage_sim.h        13 路电压模拟 + 过压/欠压/压差注入
│   │   ├── dac_safety.h         DAC 安全保护(急停/报警)
│   │   ├── waveform_gen.h       方波/正弦波发生器
│   │   ├── waveform_engine.h    高频正弦 DMA 引擎(用外部 SRAM)
│   │   ├── fault_command.h      故障命令行解析
│   │   ├── fault_console.h      本地控制台(接 Zephyr shell)
│   │   ├── tcp_json_server.h    上位机 TCP + JSON 服务(端口 5000)
│   │   ├── can_protocol.h       CAN 控制/反馈帧协议(0x011~0x021)
│   │   └── bms_response_log.h   BMS 响应延迟记录(T1/T2/ΔT)
│   └── src/<module>/<module>.c  实现(当前为骨架,由原工程搬运)
│
├── drivers/              ★换 DAC / 换收发器隔离点 —— 自定义 BSP 驱动
│   ├── dac81416/         TI DAC81416 16 通道 DAC(SPI + LDAC 同步锁存)
│   ├── can/              CAN 收发(底层接 Zephyr CAN 子系统)

│   └── sram/             外部 SRAM(FMC),波形引擎 DMA 缓冲
├── include/drivers/      BSP 驱动对外 API 头(bsp_dac81416.h / bsp_can.h ...)
│
├── dts/bindings/dac/     DAC81416 数据节点与控制引脚(RESET/LDAC/ALARM)绑定
│
├── boards/arm/           ★换 MCU 隔离点 —— 一块板一个目录
│   ├── bms_emu_h743/     STM32H743 主控板(.dts 描述 SPI/CAN/FMC/ETH/引脚)
│   └── bms_emu_altmcu/   备选板占位(换 MCU 时照模板新建)
│
├── overlays/             分场景 overlay/conf
```

## 3. 依赖方向(解耦的关键)

```
app/ (voltage_sim, waveform_engine, dac_safety ...)
   │  只 #include "drivers/bsp_dac81416.h" / "drivers/bsp_can.h"(API)
   ▼
include/drivers/*.h  ◄── drivers/dac81416, drivers/can ...(实现)
                              │ SPI/GPIO/CAN 句柄经 devicetree 获取
                              ▼
                  devicetree(.dts, boards/) + Kconfig ──► SoC HAL (hal_stm32, west 拉取)
```

- 业务层只依赖 BSP 驱动的 **API 头**,不接触任何 STM32 寄存器/HAL 类型。
- BSP 驱动的 SPI/GPIO/CAN 资源全部来自 devicetree,不写死引脚。
- 只有 devicetree 与 Kconfig 把抽象绑定到具体芯片/板卡。

## 4. 换件操作手册

### 换 DAC(如 DAC81416 → 其它)
1. `drivers/` 新增 `<newdac>/` 目录,实现 `include/drivers/bsp_dac81416.h`(或新 API 头)同名函数。
2. `dts/bindings/dac/` 加新芯片 binding,板级 `.dts` 改 DAC 节点 `compatible`。
3. `prj.conf` 关 `CONFIG_BSP_DAC81416`、开新驱动开关。
4. **`app/voltage_sim`、`waveform_engine`、`dac_safety` 一行不改**。

### 换 MCU(STM32H743 → 其它)
1. `boards/arm/` 照 `bms_emu_h743/` 新建板目录,`.dts` 用新 SoC dtsi 重写(SPI/CAN/FMC/ETH/引脚)。
2. `west.yml` 的 `name-allowlist` 把 `hal_stm32` 换成新 MCU 的 HAL。
3. devicetree 节点 `compatible`(dac81416 / dac81416-ctrl)保持不变,只改引脚/时钟/SPI 实例。
4. **`app/`、`drivers/`(除非新 MCU 外设访问方式不同)不动**。

## 5. 通信通道

- **CAN(FDCAN1)**:主控制/反馈帧,协议见 `can_protocol.h`(接收 0x011~0x017,反馈 0x01A~0x021)。
- **以太网 + TCP/JSON**:上位机指令,`tcp_json_server.h`(端口 5000)。
- WiFi:本期不实现。

## 6. 构建与测试

```bash
west init -l .          # 以本仓库为 manifest 初始化 workspace
west update             # 拉取 zephyr + hal_stm32 到 deps/
west build -b bms_emu_h743 .
```
